Implementation of Line Encoding Schemes

Pre Test

  1. Unipolar RZ is a type of ______ encoding.
    2. a. Line
    b. Block
    c. Scrambling
    d. None of the above

  2. DC component is present in the encoded signal and there is loss of synchronization for long sequences of 0’s and 1’s in unipolar RZ.
    3. a. True
    b. False

  3. NRZ-L is a type of _____ line encoding.
    4. a. Bipolar
    b. Multitransition
    c. Polar
    d. None of the above

  4. Which of the following is not a Polar line encoding scheme?
    5. a. NRZ-I
    b. NRZ-L
    c. AMI
    d. All of the above

  5. Manchester encoding is a combination of which two line encoding schemes.
    6. a. NRZ-L and RZ
    b. NRZ-I and RZ
    c. NRZ-L and AMI
    d. None of the above

  6. The ________ represents the number of bits sent per second, whereas the ______ defines the number of signal elements per second in the signal.
    7. a. bit rate, baud rate
    b. baud rate , bit rate
    c. Both are same
    d. None of the above

  7. In which line encoding scheme 0 bit is represented by Zero level and binary 1 is represented by alternating positive and negative voltages.
    8. a. NRZ-I
    b. AMI
    c. Pseudoternary
    d. All of the above

  8. In Pseudoternary line encoding
    Statement 1: 0 bit is represented by Zero level.
    Statement 2: Binary 1 is represented by alternating positive and negative voltages.
    Which of the above are true.
    9. a. Statement 1 is true and Statement 2 is false
    b. Statement 1 is false and Statement 2 is true
    c. Both are true
    d. Both are false

  9. The polarities in NRZ format use?
    10. a. Complete pulse duration
    b. Half duration
    c. Both positive as well as negative value
    d. Each pulse is used for twice the duration

  10. How many levels does manchester line encoding involve?
    11. a. Two levels with inversion in the middle of each bit
    b. Three levels with inversion in the middle of each bit
    c. Two levels without inversion in the middle of each bit
    d. Three levels without inversion in the middle of each bit
